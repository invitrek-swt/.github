# Security Policy (Invitrek-swt)

## Reporting a vulnerability
If you believe you have found a security issue:
1. Do **not** open a public GitHub issue.
2. Contact the organization maintainers privately (recommended):
   - Use GitHub Security Advisories (if enabled), or
   - Email the org security contact (to be defined by Invitrek-swt admins).

## Supported versions
Unless otherwise stated in a repository:
- Security fixes are applied to the default branch.
- Releases may be patched depending on product needs.

## Secure development basics
- Never commit secrets (tokens, passwords, certificates).
- Validate and sanitize untrusted input.
- Follow least privilege for access to data and services.
