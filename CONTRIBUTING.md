# Contributing (Invitrek-swt)

## Purpose
This document defines the default contribution process for repositories under the `Invitrek-swt` organization.

## Basic rules
- Keep changes small and focused.
- Follow organization coding standards (see `Invitrek-swt/engineering-standards`).
- No secrets or credentials in code, issues, or PRs.

## Workflow
1. Create a branch from `main` (or the repo default branch).
2. Implement changes with tests.
3. Run build/tests locally.
4. Open a Pull Request (PR) using the PR template.
5. Address review comments and ensure CI passes.

## Pull request expectations
- Clear description of what changed and why
- Test coverage included/updated
- Screenshots for UI changes (where applicable)
- Migration notes if data/contracts change

## Reporting issues
Use the issue templates for bug reports and feature requests.

## Security
For security issues, follow `SECURITY.md` and do not disclose details publicly.
