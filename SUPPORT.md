# Support

## Getting Help

There are several ways to get help with any **invitrek-swt** project:

### 1. Documentation

Before reaching out, please check:
- The project's `README.md`
- The project's `docs/` directory
- Organization-wide standards in [invitrek-swt/.github](https://github.com/invitrek-swt/.github)

### 2. GitHub Discussions

For questions, ideas, and general discussion, open a [GitHub Discussion](../../discussions) in the relevant repository. Discussions are the preferred channel for:
- Usage questions
- Design discussions
- Brainstorming ideas

### 3. GitHub Issues

Use [GitHub Issues](../../issues) only for:
- Confirmed bugs (use the **Bug Report** template)
- Feature requests (use the **Feature Request** template)

Do not use issues for general support questions — use Discussions instead.

### 4. Security Issues

**Do not report security vulnerabilities in public issues.** See [SECURITY.md](SECURITY.md) for the responsible disclosure process.

---

## Response Expectations

| Channel | Expected response time |
|---------|----------------------|
| Security reports | 2 business days |
| Bug reports | 5 business days |
| Feature requests | 10 business days |
| Discussions | Best effort |
